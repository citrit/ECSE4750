  • How does our example fit into the functional model?
  • What does the Source object have to do in order to generate the appropriate information?
  • What kind of methods are necessary for these objects to both generate geometry and render it?
  • In general we can think of these objects into 2 groupings:
    1. Data Creation objects: They are responsible for generating the underlying data for the geometry primitives. This behavior is encapsulated in an Execute() method. If you want to create a new source object, you do not need to make it a subclass of any convenience object but it should be derived from vtkSource.
    2. Geometry Display objects: Mappers map its input dataset to the appropriate rendering library/system. Subclasses of the vtkMapper class must implement a Render() method. This method is exchanged by the graphics systems actors and its associated mapper during the rendering process.
  • This allow the system to differentiate between generation and display of data/geometry.
