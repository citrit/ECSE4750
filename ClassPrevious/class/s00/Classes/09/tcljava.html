Tcl Java Integration

Download | New Features | FAQ | Man Pages

Tcl & Java integration currently consist of two systems, Jacl and Tcl Blend. Jacl, which stands for Java Command Language, is a new Java implementation of Tcl 8.0. Tcl Blend is a new package for Tcl 8.0 that allows you to load and interact with the Java VM.

With the advent of these systems, you can write extensions for Tcl, in Java code, that will be cross platform across UNIX, Windows and the Macintosh. In addition, you can use a new set of Tcl instructions that allow you to create, call and manipulate Java objects directly from Tcl. These new API's are ideal for rapid prototyping and testing of Java interfaces. Using this interface provides an easy way to get access to Java's powerful libraries without having to leave Tcl.

Road Map

  • Jacl & Tcl Blend version 1.0: The 1.0 release of Jacl and Tcl Blend were made available from Sun Laboratories February 18, 1998. This was the final release of the 1.0 version and the conclusion of the work being done by the Sun team.

  • Jacl & Tcl Blend version 1.1.1: A new 1.1.1 release of both Jacl and Tcl Blend were made available for downloading on March 6, 1999. This release is due entirely to the efforts of Christopher Hylands and Mo DeJong, with minimal assistance from Scriptics. This release fixes numerous bugs in the 1.0 release and adds some new features that will make combining Tcl and Java even easier then before. More information about this release can be found on the New Features page.

  • Jacl & Tcl Blend 1.1 Patch Releases: New features and bug fixes will show up in the 1.2 tree. If a serious bug is found and there is lots of user interest, the bug fix may be back-ported to the 1.1 tree.

  • Jacl & Tcl Blend 1.2 And Beyond: Unfortunately, Christopher Hylands has moved on to other tasks, and will only provide minimal support for future releases of Jacl and Tcl Blend. However, Mo DeJong is aggressively working on the 1.2 release of Jacl and Tcl Blend. His work is available from his own Development Branch.

Why Jacl?

  • 100% Java!: Jacl is completely implemented in Java. Tcl has always been cross platform, but having a version of Tcl implemented in Java allows Tcl code to run on even more platforms including the Java Station and many OSes that Tcl has not been ported to.

Why Tcl Blend?

  • Porting legacy code: Tcl Blend offers an easy way to create hybrid applications made from both legacy code and Java code. It's a great way to slowly migrate legacy applications to Java over a period of time.

  • Java integration at the speed of C: As an extension package for Tcl8.0, Tcl Blend offers Tcl/Java integration without compromising the efficiency of core Tcl features.

Additional Information

  • A white paper that describes the benefits of Tcl & Java integration was written by Ray Johnson on January 29, 1998. Ray Johnson was the Project Lead for the 1.0 release at Sun Laboratories. His paper provides a high-level overview of the motives and goals for Tcl & Java integration. The paper is available for downlaod in postscript or pdf formats.

Jacl & Tcl Blend 1.1.1 Known Restrictions

  • Applets: Jacl will not currently work inside of Browsers. The reason is that all of the browsers we tested against had very weak support for the JDK 1.1 features that Jacl depends on. In addition, there has been limited interest in running Jacl in a web browser so this feature has received little attention.

  • Jacl: Unfortunately, Jacl does not yet contain the entire Tcl feature set. Jacl includes all of the control structures and a majority of core Tcl commands. There are plans to implement namespaces and sockets but they have not been implemented as yet. On Windows, you must use Sun's JDK because Microsoft's Java implementation does not properly implement IO features that Jacl depends on.

  • Tcl Blend: Currently, Tcl Blend has only been tested on Solaris and Windows NT. Tcl Blend requires JDK 1.1, or newer, and Tcl 8.0. Tcl Blend will not compile under Tcl 8.1 at this time. On Windows, you must use Sun's JDK because Microsoft's Java development kit omits JNI, which is a necessary interface for Tcl Blend.

  • Regular Expressions: A regular expression package licensed from a third party is included in Jacl. This package provides almost all of regular expression functionality of Tcl. Some regular expressions may differ slightly between Jacl and regular Tcl, but these are the exception and not the rule. The source code to the regular expression library is not provided with the Jacl release but the binary .class files needed to run regular expressions in Jacl are provided.
