Using a GUI to design a GUI

  • So far, we've had to write a program to create a GUI.
  • A GUI interface that lets us drag and drop widgets might be easier.
  • Of course, the limitation with the WYSIWYG (What You See Is What You Get) metaphor is that it is also What You See Is All You Get.
  • That is, it's hard to automate repeated actions, and to store deeper meaning, or even a comment, in the document.
  • Nevertheless, these tools do make it easier to get started.
  • Most such tools are platform-specific, proprietary, expensive, or all 3 simultaneously.
  • Here's one written in Tcl. Vtcl is free and portable.

Vtcl - Visual Tcl

The following is modified from http://www.neuron.com/stewart/vtcl/, as permitted by the copyleft.

Overview

Visual Tcl is a freely-available, high-quality application development environment for UNIX, Windows, Macintosh and AS400 platforms. Written entirely in Tcl and generating pure Tcl makes porting either unnecessary or trivial. Visual Tcl is covered by the GNU General Public License and has no relation to SCO Visual Tcl.

The author is Stewart Allen.

Product Features

  • 100% pure Tcl/Tk. No external libraries required.
  • Extensible widget and geometry manager support.
  • Create compound widgets and widget libraries.
  • GUI interface for most aspects of Tcl/Tk development.
  • Imports pre-existing Tcl/Tk code.
  • Exports Tclets which run in Netscape/MSIE.
  • Here is a page, http://www.neuron.com/stewart/vtcl/snapshot.html, which shows the various toolbars.
