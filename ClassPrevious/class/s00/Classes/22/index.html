  15. Animate some data structure or algorithm. Show how a sort routine works, or how you rotate a tree to balance it.
  16. A recent CACM had a paper on verifying a user's identity by measuring the time between keystrokes as he types his name. Test this. The graphical input might be assorted options that you would set with sliders etc.
  17. Do a psychological experiment on users --- how well they can match colors in different color spaces, how easily they can use different types of physical devices for the same logical device, etc.
  18. Experiment with different dithering methods. Extend this to a tutorial. Compare the times to implement and times to execute of several different circle algorithms.
  19. Make stereo pictures, photo them on slide film, and put them in a ViewMaster.

Digital Elevation Models of the US

A DEM is a grid of 1201x1201 elevations on a 3-arc-second spacing (about 90m) for a 1 degree by 1 degree cell of the coterminous US. See here for more info. There is a sample file, of the Lake Champlain West topo quad, online.

I can put another two dozen files online. You can also FTP all the US DEMs from the United States Geological Survey, altho then you have to decode their format. There are Windows programs that do that.

Voronoi Diagrams

Consider a set of points representing firehouses. Divide the city into a region around each firehouse, representing that part of the city closer to that firehouse than to any other. That is called the Voronoi diagram of the points. Xvor is a program to interactively place points, give the velocities, and watch the changing Voronoi diagram. The Sun executable is n dir sun.

Here is a sample output. The point are where the solid lines meet. (The solid lines triangulate the points' convex hull.) The Voronoi diagram on the points is the dotted lines.

POV Ray

POV-Ray is a neat ray-tracing program. You define objects in terms of cylinders, cubes, etc, with their colors and surface properties, such as shininess, and the light sources. POVray calculates the image. POVray's home site is http://www.povray.org/.

POVray is in /dept/acm/packages/povray/2.2. There is a doc file, which is 52 pages if printed 2-up with enscript thus: 

Docs/povray.doc | enscript -2r -Pvclw

First, you create a file defining the scene, such as pov/a.pov, which I got from /dept/acm/packages/povray/2.2/Docs/demo/arealit1.pov.gz. 

// Persistence Of Vision raytracer version 2.0 sample file.
// Illustrates how area light grid size affects soft shadows

// Left   shadow - area_light <8,0,0>, <0,0,8>,  3,  3 (renders fastest)
// Middle shadow - area_light <8,0,0>, <0,0,8>,  5,  5
// Right  shadow - area_light <8,0,0>, <0,0,8>, 17, 17 (renders slowest)

#include "colors.inc"
#include "textures.inc"

// A back wall to cast shadows onto
plane { -z, -20
    pigment { Gray70 }
    finish { Dull }
}

#declare SpacingX = 20
#declare Radius = 5
#declare LightX = 15
#declare LightY = 40
#declare LightZ = -40
#declare SRadius = 0
#declare SFalloff = 11

#declare SphereTexture = texture {
    pigment { Red }
    finish { Shiny }
}

sphere { <-SpacingX, 0, 0>, Radius
    texture { SphereTexture }
}

light_source {
    <0, LightY, LightZ> color White
    area_light <8, 0, 0>, <0, 8, 0>, 3, 3
    adaptive 0
    jitter

    spotlight
    point_at <-SpacingX, 0, 0>
    tightness 0
    radius SRadius
    falloff SFalloff
}

sphere { <0, 0, 0>, Radius
    texture { SphereTexture }
}

light_source {
    <0, LightY, LightZ> color White
    area_light <8, 0, 0>, <0, 8, 0>, 5, 5
    adaptive 0
    jitter

    spotlight
    point_at <0, 0, 0>
    tightness 0
    radius SRadius
    falloff SFalloff
}

sphere { <+SpacingX, 0, 0>, Radius
    texture { SphereTexture }
}

light_source {
    <0, LightY, LightZ> color White
    area_light <8, 0, 0>, <0, 8, 0>, 17, 17
    adaptive 0
    jitter

    spotlight
    point_at <+SpacingX, 0, 0>
    tightness 0
    radius SRadius
    falloff SFalloff
}

light_source { <0, -15, -120> color Gray10 }

camera {
    location <0, -15, -120>
    direction 2*z
    look_at <0, -15, 0>
}

Then you run POVray thus: 

cp /dept/ecse/graphics/h10/pov/a.pov .
povray +Ia.pov +Oa.qrt +V +W400 +H400 +FD +L/dept/acm/packages/povray/2.2/common/include

to get this image showing 3 balls casting shadows from 3 lights.

For previewing, I recommend doing a 60x60 image; I did 400x400 just to make the printed image look better, but it took 11 1/2 minutes on my Sun IPC, or 2 CPI minutes on a Sun 10/30.

The output file, a.qrt, has to be converted so that xv can read it, thus: 

qrttoppm a.qrt >a.ppm 
xv a.ppm

qrttoppm is in /campus/other/netpbm/1mar94/\@sys/bin/qrttoppm. There were a number of sample command files and resulting scenes, in /dept/acm/packages/povray/2.2/distrib/scenes/, but they took too much space and are gone now.

(c)1995 William J. Beaty

I've stumbled across a technique for drawing holograms directly upon a plastic plate by hand. It sounds crazy, but I've been sitting on the livingroom sofa making holographic images of floating polyhedra, words, 3D starfields, etc. No laser, no isolation table, no darkroom, no expensive film plates. This takes nothing more than a compass and some scraps of black plexiglas. Too cool, if I say so myself!.....

Color summary

This logically follows on from hardware.

  1. Refs:

    1. Hearn & Baker, chapter 15.

    2. efg's Computer Lab Reference Library: Color, at http://www.efg2.com/lab/library/Color.htm

  2. Visible light comes in wavelengths from 400 (violet) to 700nm (red). 1nm=10-9 meters.

  3. The 3 types of cone cells in your retina sense approximately red, green, or blue.

  4. Since we projecting down from an infinite-dimensional space (all the wavelengths from 400 to 700 nm) to a 3-D space (R,G,B), there is an enormous loss of info. Two objects that look the same color might be emitting or reflecting quite different wavelengths. This is why two objects can look the same color under an incandescent lamp, but look different in daylight.

  5. The CIE chromaticity diagram, page 570, http://www.nd.edu/~sboker/ColorVision2/CIEColorSpace2.gif

    maps a wavelength to an (x,y) point. It is designed so that when you combine two colors, you'll know what the result looks like. (x1,y1) mixed with (x2,y2) looks like ((x1+y1)/2,(y1+y2)/2).

    Designing this curve was not as easy as it sounds: what does a mix of violet and green light look like? It was based on actual human tests.

  6. The CIE coordinates are really 3D: (x,y,z). The diagram just normalized the brightness.

  7. You can easily convert back and forth:

    and

  8. For things that emit light, like CRTs, we talk about additive colors. The primaries are red, green, and blue (RGB space). We can display only colors inside the triangle formed by the 3 phosphors we're using (page 573).

  9. For things that reflect light, like paper, we talk about subtractive colors. The primaries are cyan, magenta, and yellow (CMY space). We can display only unsaturated colors; the more saturated an ink is, the darker it is.

  10. There are other color models, like HSV (hue, saturation, value), and HLS (hue, lightness, saturation); see the text. They try to be more intuitive.
