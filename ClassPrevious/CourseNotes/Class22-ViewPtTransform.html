The Viewpoint Transform

  • An implementation of this would be to add transforms to the CGWindow class to handle various parts of the Viewing Pipeline
  • We will add:
    1. m_WorldToViewTransform to handle the World to View corrdinate shift.
    2. m_ViewToDisplayTransform to handle the translation from View coordinates to Display points.
  • Here is the example implementation: hw8ans/cgxwindow.hhw8ans/cgxwindow.cpp
  • This handles translating a window in world coordinates into display coordinates, bypassing the normalized coordinates.
  • First we specify the Window in world coordinates we wish to see:
    • /** Set the World coordinates for the window */
void CGWindow::SetWindow(const CGPoint pt1, const CGPoint pt2)
{
        m_WindowRect.SetRect(pt1, pt2);
        CalculateViewTransform();
        SetSize(m_Size[0], m_Size[1]);
}
  • This translates the Word Coordinates to the views 0,0 origin and calls the SetSize method to ensure the ViewToDisplayTransform is also updated.
  • We could also add a method to Rotate the Window:
    • /** Rotate the window about the origin by the specified angle */
void CGWindow::SetWindowAngle(double angle)
{
        m_WindowRotation = angle;
        CalculateViewTransform();
}
  • This method sets the internal rotate angle and calculates the ViewTransform.
  • The CalculateViewTransform method looks like:
    • /** Use the member variable to define WorldToView transform */
void CGWindow::CalculateViewTransform()
{
        const CGPoint pt1 = m_WindowRect.GetLowerLeft();
        CGPoint pt2 = m_WindowRect.GetUpperRight();
        m_WorldToViewTransform.Reset();
        m_WorldToViewTransform.Translate(-pt1.x, -pt1.y);
        pt2.x = (pt2.x - pt1.x)/2.0 + pt1.x;
        pt2.y = (pt2.y - pt1.y)/2.0 + pt1.y;
        m_WorldToViewTransform.Rotate(m_WindowRotation, pt2);
}
  • Things to make sure of:
    1. What happens when the user resizes the window?
    2. What do we need to do when a Expose event happens.

  • To complete homework #8 you needed to understand the collection based metality of our graphics system.
  • Objects are defined in the file but you know little of what type they are, how many, or the size of any.
  • The trick is to read in the file and create objects as necessary.
  • Then collect them into the appropriate container.
  • Here is the code implemented for homework 8: hw8ans
  • The key is to make the CGWindow have methods that are easily used by the shape classes
  • As you can see the shape classes are very simple but allow a great deal of customizing.
  • The core window code needs to understand actors and order of appearance (i.e. who needs to be sent a message first).
  • The ViewTransforms need to be specified first prior to any drawing.
  • Updates and caching needs to be implicitly determined.
