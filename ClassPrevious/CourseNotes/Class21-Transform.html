Term Projects:

  • What I am looking for in the term projects is a new way to look at old issues.
  • Try not to fall into the old thinking model. Take an old problem and create a new fresh paradigm.
  • Computer graphics naturally preceeds the need or want to visualize your data or system.
  • In general terms, Scientific Visualization can be thought of as any method which presents scientific information in a manner to facilitate the conceptualization of scientific phenomenon or statistical information. With advances in computational speeds and display technology, sci-vis has experienced a revolution. Historical methods of sci-vis have not become obsolete however. Graph paper and oscilloscopes have been key sci-vis tools for years, and continue to be so. Scientific visualization may take advantage of any of the human senses. Sound has been used as a sci-vis tool for years and is still an area of active research. New techniques are being studied which employ the sense of feel, including force-feedback systems to help "feel" the strength of a bond between atoms, or feel the molecular surface of a tooth.

Color:

  • We discussed adding per-face coloring as well as per-object coloring.
  • This can be accomplished by adding a collection to the Actor to gather a color for each shape.
  • Then we apply the colors in the order of the Added Shapes.
  • The issue arises when there are not enough colors for all the shapes.
  • We will fix this by modulating on the total number of colors. A modified CGActor::Draw method:
    •        /** Time to draw */
       void CGActor::Draw(CGWindow *cgw) {
        unsigned int ii, numColors(1);
        
        if (m_vObjectColors.empty()) {
                CGColor cgc;
                m_vObjectColors.push_back(cgc);
        }
        else {
                numColors = m_vObjectColors.size();
        }
        
        for (ii=0;ii<m_vShapes.size();ii++) {
                cgw->SetColor(m_vObjectColors[ii%numColors]);
                m_vShapes[ii]->Draw(cgw, m_Trans);
        }
      }
  • We also need  to change the CGActor's definition of m_ObjectColor from CGColor to vector<CGColor &>.
  • The CGActor::SetColor method now becomes CGActor::AddColor.
    •   void AddColor(const CGColor &cgc) { m_vObjectColors.push_back(cgc); this->Modified(); }

The Viewpoint Transform

  • We need a method to define what part of our scene is to be displayed, a Window.
  • And we need to define the area of the screen to display this on, the Viewport.
  • Our task is to map the Window to the Viewport on the screen.
  • Initially this seems relatively easy, we can transform and scale the window to match our viewport coordinates.
  • One twist is to add the ability to rotate the window and display the results on the output device.
  • Also the viewport will be defined within the unit square, or we will use normalized coordinates. This ensures the transformation is device independant.
  • The transformation is accomplished by specifying a view-coordinate origin Po = (xo, yo)
  • We then specify the orientation using a world vector V that specifies the viewing yv direction. This is called the view up vector.
  • Given V we can calculate the components of the unit vectors v = (vx, vy) and u = (ux,uy).
  • We will use these vectors as the first and second rows of our rotation matrix R that aligns the viewing xv,yv axis with the world xw,yw axes.
  • We can create the matrix for converting the world-coordinate positions to viewing coordinates as a two-step composite transformation:
    1. We translate teh viewing origin to the world origin
    2. We rotate to align the two coordinate reference frames.
  • The composite two-dimensional transform is :

      • Mwc,vc = R.T

  • T is the translation matrix used to translate the viewing point P to the world origin
  • R is the rotation matrix that aligns the two coordinate systems.
  • Now we need to map these coordinates to normalized device coordinates.
  • We do this using a transformation that maintains the same relative placementof objects in normalized space as they had in viewing coordinates.
  • If a coordinate point is in the middle of the viewing window, it will map to the center of the viewport.
  • This is done by using the max and min of the viewing window and viewport to normalize coordinates.
      • XV = XVmin + (XW - XWmin)sx
YV = YVmin + (YW - YWmin)sy
  • where the scaling factors are
      • sx = (XVmax - XVmin) / (XWmax - XWmin)
sy = (YVmax - YVmin) / (YWmax - YWmin)
  • This can be derived with a set of transformations that convert window area to viewport area

    1. Perform a sclaing transform using a fixed-point position of (XWmin,YWmin) that scales the window area to the size of the viewport
    2. Translate the scaled window area to the psoition of the viewport.

  • Relative proportions of objects are maintained as long as sx = sy.
